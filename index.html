Sublime3上Markdown插件-MarkdownEditing的安装和用法(译)

发表于 2017-02-08   |     |   阅读次数

本文基于:
系统：Mac os
软件：Sublime text3
插件：MarkdownEditing

安装

通过Package Control

  • 首先安装Package Control。
  • 在sublime中，使用快捷键 CMD+SHIFT+P 打开Package Control。
  • 在弹出的输入框中输入 install package 然后点击匹配项。这时候一个可用的包列表将会显示。
  • 在可用列表的输入框中输入 MarkdownEditing 然后点击匹配项。该插件将会被下载。
  • 重启Sublime完成安装。

特点

快捷键

  • command + option + v:创建或者粘贴剪切板上的内容作为选中文本的内联链接
  • command + option + r:创建或者粘贴剪切板上的内容作为引用链接
  • command + shift + k:创建或者粘贴剪切板上的内容作为选择文本的内联图片
  • command + option + b/i:b代表粗体，i代表斜体。无论选不选中它们都会产生作用，如果没有选中，它们只会转变光标下的词，如果选中的字体已经加粗或者斜体，使用该功能会取消选种字体的加粗或者斜体。
  • command + control + 1…6:
  • option + shift + 6:添加脚注
  • command + tab:折叠或者展开当前部分(此快捷键和mac自带切换程序快捷键冲突)
  • command + shift + tab:折叠一定层级下的所有部分
  • command + option + pageup/pagedown:去当前/下一个的同等级或者更高等级的heading
  • command + shift + pageup/pagedown:去当前/下一个heading

网络相关知识点总结

发表于 2017-02-08   |     |   阅读次数

名词介绍

  • WWW：world wide web（万维网）
  • HTML：HyperText Markup Language（超文本标记语言）
  • HTTP：HyperText Transfer Protocol（超文本传输协议）
  • URL：Uniform Resource Locator（统一资源定位符）
  • URI：Uniform Resource Identifier（统一资源标识符）
  • FTP：File Transfer Protocol（文件传输协议）
  • DNS：Domain Name System（域名系统）
  • TCP：Transmission Control Protocol（传输控制协议）
  • UDP：User Data Protocol（用户数据包协议）
  • NIC：Network Interface Card（网卡）
  • IP：Internet Protocol（网际协议）
  • MAC：Media Access Control Address（物理地址）

Source Code（状态码）

2xx

  • 200 OK：请求正常处理
  • 204 No Content：请求正常处理，没有内容返回
  • 206 Partial Content：范围请求

3xx

Method

  • GET：获取资源
  • POST：修改资源
  • PUT：上传文件（一般不适用该方法，因为自身没有验证机制，任何人都可以上传，存在安全性问题）
  • DELETE：删除文件，一般不用，原因同上

HTTP三次握手

第一次握手：client发送SYN包给server，进入SYN_SEND状态，等待server确认
第二次握手：server收到SYN包，确认收到客户的SYN包，同时自己发送一个SYN+ACK包，此时服务器进入SYN_RECV状态
第三次握手：client收到server的SYN+ACK包，向服务器发送确认ACK，此包发送完毕，client和server进入ESTABLISHED状态，完成三次握手。

mysql总结

发表于 2017-01-23   |     |   阅读次数

系统级命令

  • mysql -u root -p
  • brew services start mysql:启动mysql服务
  • brew services stop mysql:停止mysql服务
  • exit:退出mysql环境

SQL基础语句

DDL

  • CREATE DATABASE databasename- 创建新数据库
  • ALTER DATABASE - 修改数据库
  • CREATE TABLE tablename- 创建新表
  • DROP TABLE tablename - 删除表格
  • ALTER TABLE - 变更（改变）数据库表
  • DROP TABLE - 删除表
  • CREATE INDEX - 创建索引（搜索键）
  • DROP INDEX - 删除索引

DML

  • 增
    INSERT INTO Persons VALUES (2, ‘Gates’, ‘Bill’, ‘Xuanwumen 10’, ‘Beijing’)

  • 删
    DELETE FROM 表名称 WHERE 列名称 = 值
    DELETE FROM Person WHERE LastName = ‘Wilson’
    DELETE FROM table_name or DELETE * FROM table_name:可以在不删除表的情况下删除所有的行。这意味着表的结构、属性和索引都是完整

  • 改
    UPDATE 表名称 SET 列名称 = 新值 WHERE 列名称 = 某值
    UPDATE Person SET FirstName = ‘Fred’ WHERE LastName = ‘Wilson’

  • 查
    *代表所有的列
    SELECT username FROM userinfo

where
SELECT * FROM Persons WHERE City=’Beijing’

and
select * from userinfo where username=’fzh’ and departname=’dev’

distinct
SELECT DISTINCT Company FROM Orders :去重查找

ORDER BY :默认为升序，DESC为降序
select username from userinfo order by created

SQL高级语句

  • top:限制返回数据的数量
    SELECT username FROM userinfo LIMIT 3

  • LIKE 操作符用于在 WHERE 子句中搜索列中的指定模式
    select * from userinfo where username like ‘f%’

通过使用 NOT 关键字来表示否
select * from userinfo where username not like ‘%a%’

  • IN 操作符允许我们在WHERE子句中规定多个值
    select * from userinfo where username in (‘aaa’,’fzh’)

  • BETWEEN … AND 会选取介于两个值之间的数据范围。这些值可以是数值、文本或者日期
    select username from userinfo where username between ‘aaa’ and ‘bbb’

请使用 NOT 操作符来表示否
select username from userinfo where username not between ‘aaa’ and ‘bbb’

  • NOT NULL 约束强制字段始终包含值。这意味着，如果不向字段添加值，就无法插入新记录或者更新记录

  • UNIQUE 约束唯一标识数据库表中的每条记录,每个表可以有多个 UNIQUE 约束，但是每个表只能有一个 PRIMARY KEY 约束

相关配置文件说明

  • /etc/profile（建议不修改这个文件）全局（公有）配置，不管是哪个用户，登录时都会读取该文件。

  • /etc/bashrc(一般在这个文件中添加系统级环境变量)全局(公有)配置，bash shell执行时，不管是何种方式，都会读取此文件。

  • ~/.bash_profile(一般在这个文件中添加用户级环境变量)每个用户都可使用该文件输入专用于自己使用的shell信息,当用户登录时,该文件仅仅执行一次!

  • 想配置文件立即生效： 运行source .bash_profile

    切换shell

  • 若为bash，在终端输入zsh，则切换为zsh

  • 若为zsh，在终端输入bash，则切换为bash

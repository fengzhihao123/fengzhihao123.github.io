开篇

今天在修改项目的时候，看见enum中出现了<<操作符(位操作)，之前对这个一直都不了解。这次趁着项目比较清闲，抽出时间来全面了解一下位操作。

位操作

位操作是对二进制数逐位进行运算或移位。它共包含两种操作：位运算和移位。下面就详细的了解一下这两种操作。
在此只讨论iOS中的所有位操作的运算符，别的语言的相同含义的操作符号可能不同

位运算符(以下操作符皆同Objective-C)

位运算符一种包含下面几种：

  • ~(取反,一元操作符)：它会对目标数字的二进制每位进行取反

  • |(按位或)：它会对两个目标数字的相同位置数字进行或运算，规则：0和0为0；0和1为1；1和1为1

  • &(按位与)：它会对两个目标数字的相同位置数字进行与运算，规则：0和0为0；0和1为0；1和1为1

  • ^(异或)：它会对两个目标数字的相同位置数字进行异或运算，如果不同则该位为1，否则该位为0。规则：如0和0为0；0和1为1；1和1为0

移位

  • >>(右移)：它会对目标数字按位右移x位

  • <<(左移)：它会对目标数字按位左移x位(右边补0)

枚举中的位操作

通过上文我们了解了位操作的具体计算方式，接下来看一下在枚举中的具体应用。

枚举中的应用

定义枚举

  • OC

  • Swift

位操作在枚举中的作用

  • ~(取反)：用来剔除某个值

  • |(按位或)：用来添加某个值

  • &(按位与)：用来检查是否包含某个值

  • ^(异或)：用来置反某个值(如果包含则剔除，如果不包含则添加)

在枚举中使用位操作我们可以方便的给一个属性值赋值多个值，比如下面的代码给lineType赋值了CellExLineTypeTopNone和CellExLineTypeBottomNone属性。这样我们就可以在lineType的set方法里面处理CellExLineTypeTopNone和CellExLineTypeBottomNone的情况。

在系统中的许多Enum也是这么使用的，如UIViewAutoresizing、UIViewAnimationOptions等。

为什么要在枚举中使用位操作符?

  • 在许多古老的微处理器上，位运算比加减运算略快，通常位运算比乘除法运算要快很多。在现代架构中，情况并非如此：位运算的运算速度通常与加法运算相同（仍然快于乘法运算）
  • 可以给一个属性同时设置多个值

    总结

  • ~(按位取反)：对目标数字按位取反；在枚举中用于剔除某个值
  • |(按位或)：对两个目标数字同位置上数字进行或运算；在枚举中用于添加某个值
    • &(按位与)：对两个目标数字同位置上数字进行与运算；在枚举中用于判断是否包含某个值
  • ^(按位异或)：对两个目标数字同位置上数字进行异或运算；在枚举中置反某个值
  • >>(右移)：对目标数字按位右移x位
  • <<(左移)：对目标数字按位左移x位

网络相关知识点总结

发表于 2017-02-08   |     |   阅读次数

名词介绍

  • WWW：world wide web（万维网）
  • HTML：HyperText Markup Language（超文本标记语言）
  • HTTP：HyperText Transfer Protocol（超文本传输协议）
  • URL：Uniform Resource Locator（统一资源定位符）
  • URI：Uniform Resource Identifier（统一资源标识符）
  • FTP：File Transfer Protocol（文件传输协议）
  • DNS：Domain Name System（域名系统）
  • TCP：Transmission Control Protocol（传输控制协议）
  • UDP：User Data Protocol（用户数据包协议）
  • NIC：Network Interface Card（网卡）
  • IP：Internet Protocol（网际协议）
  • MAC：Media Access Control Address（物理地址）

Source Code（状态码）

2xx

  • 200 OK：请求正常处理
  • 204 No Content：请求正常处理，没有内容返回
  • 206 Partial Content：范围请求

3xx

  • 301 Moved Permanently:永久性重定向
  • 302 Found：临时性重定向
  • 303 See Other：请求对应的资源存在另一个URI，应该使用GET定向获取资源
  • 304 Not Modified
  • 307 Temporary Redirect

4XX

  • 400 Bad Request：请求报文中存在语法错误
  • 403 Forbidden：服务器拒绝对请求资源的访问
  • 404 Not Found：没有找到资源或服务器拒绝访问

5XX

  • 500 Internal Server Error
  • 502 Bad Gateway
  • 503 Service Unavailable

Method

  • GET：获取资源
  • POST：修改资源
  • PUT：上传文件（一般不适用该方法，因为自身没有验证机制，任何人都可以上传，存在安全性问题）
  • DELETE：删除文件，一般不用，原因同上

HTTP三次握手

第一次握手：client发送SYN包给server，进入SYN_SEND状态，等待server确认
第二次握手：server收到SYN包，确认收到客户的SYN包，同时自己发送一个SYN+ACK包，此时服务器进入SYN_RECV状态
第三次握手：client收到server的SYN+ACK包，向服务器发送确认ACK，此包发送完毕，client和server进入ESTABLISHED状态，完成三次握手。

  • json web token

读书之路-读《蔡康永说话之道上》有感

发表于 2017-01-15   |     |   阅读次数

前言

从学校不如社会，最大的改变就是环境变得更加的复杂。在学校里面与别人交流比较单纯，不用想太多有的没的，但是步入社会之后，你会发现想得简单就会被套路。加上我又是说话比较不经大脑的人，所以经常被套路。希望通过读书多学一些东西，能够跟别人更好的交流，少被套路。

说明

《蔡康永的说话之道》这本书分上下两部，每部定价30元，第一本是总结出40条的核心思想，然后在通过核心语句来扩展描述。我读上部这本书大概花了8个小时左右，读完这本书之后，个人感觉没有值回书钱，感同身受的比较少(纯属个人感觉，不喜勿喷)。下面我将个人觉得比较好的分享一下。

个人感觉不错的章节

  • 外表好不好看，绝对不是人生的决胜点

现在的社会过于浮躁，致使重视心灵和内在的比较少，导致大众过多的重视外貌。所以才导致现在多家整容整形医院的存在，虽然不乏各种整形失败甚至死亡的案例，依然抵挡不住群众整形的热潮。其实当你与别人相处的时候，好的容貌确实能给你带来好的第一印象，但是如果你的性格存在各种缺陷，别人也只是会说，别看那个人长得人模狗样的，但是人品差的一逼。这样的话，别人照样不会和你做朋友的。

  • 沉默是没问题的，沉默是很正常的

你有没有曾经和你不熟的人在一起时，常常陷入没话说的尴尬境界中，因为我本身就是跟熟的的人能胡扯一天，但是跟不熟的人就说不话的人，所以我跟不熟的人坐在一起没话说的时候总会感觉很尴尬。其实想想完全没必要，享受沉默也是不错的，既然双方没有那么熟，何不安静的呆一会。

  • 不想交谈言深的话，应该避开地雷

与刚认识的人或者跟想认识的交流的时候，不太适宜交谈一些损人或者容易触及别人底线的东西。例如书中举得例子，在你上某个教授的课时，你看见邻座的女生非常漂亮，这时候你上去搭讪说:“你看那个教授长得好奇怪，长得跟马铃薯似得”。结果女生说了一句，那是她爸就走了。这他妈的就很尴尬了。所以当你第一次跟别人交流的时候最好说一些不带有攻击性的话题。

  • 交谈不是有奖金的竞赛，别急着抢答

在高手面前，装逼是没有用的，只会让别人看到你更多的缺点，最好的就是珍惜碰到高手的机会，好好地学习高手为什么成为高手。在与别人的交流中，不要打断别人正在讲的话，比如别人正在讲一个你听过的冷笑话，你应该适当的配合一下，而不是将答案脱口而出，我相信这样并不会帮助你成为别人的朋友。(以前我也是爱打断别人说话的人，现在不是了)

  • 初次见面，应该说些什么

正如文中所讲，初次见面，如果对方说了自己的名字，你也应该自报家门，这是最起码的礼貌。

  • 赞赏，观察对方最想被认可的部分

我在很多不同的书上都提到了，赞赏比批评更加有用，你批评别人，只会让别人对此事产生抵触的情绪(当然并不是所有的都不能批评，这是就事而论的)。

  • 别人赞美你一句，你就回一句赞美

  • 不要群发祝福短信

群发短信这种东西并不能巩固你跟接受者的友情，你这样反而会使对方感觉你们的距离更加的遥远。在发祝福短信的时候最起码应该在前面加上对方的名字，让对方知道你不是在敷衍这份关系。

  • 炒热气氛，需要练习(不要在KTV当麦霸)

在有些聚会中，往往有些人会肆无忌惮顶替别人的歌或者拿着话筒一直唱到结束，大家都是来放松的，不是来听你的个人演唱会的。

个人不同意的一些章节

  • 带来惊叹号，就会留下深刻的印象

本章节书中讲到，当所有人都照本宣科的时候，谁能带来惊叹号，谁就能留下深刻的印象，虽然未必是好印象，但比起完全忽略来讲，还是比较好的事吧。
但是这种观点也导致各种标题党横飞，各种无营养的新闻和娱乐人为了赚眼球而搏出位。做人做事，我觉得还是要有点底线和良心的，虽然良心并不能让你赚钱。

总结

任何的说话技巧都不是万能的，面对不同的人要有不同的方式。而且要在现实交流中不断的总结反省自己，只有多和别人交流才能知道自己的不足在什么地方。
用本书的一句话结尾。
我觉得语言最美的时候，就是在我们透过语言感受到彼此相互需要的时候

1.package

package （在我们的例子中是package main）这一行告诉我们当前文件属于哪个包，而包名main则告诉我们它是一个可独立运行的包，它在编译后会产生可执行文件,Go使用package（和Python的模块类似）来组织代码。main.main()函数(这个函数位于主包）是每一个独立的可运行程序的入口点.

Tip

  • map是无序的，每次打印出来的map都会不一样，它不能通过index获取，而必须通过key获取
  • map的长度是不固定的，也就是和slice一样，也是一种引用类型
  • 内置的len函数同样适用于map，返回map拥有的key的数量
  • map的值可以很方便的修改，通过numbers[“one”]=11可以很容易的把key为one的字典值改为11

  • map和其他基本型别不同，它不是thread-safe，在多个go-routine存取时，必须使用mutex lock机制

strings package

  • strings.HasPrefix(s, prefix string) bool: 判断前缀
  • strings.HasSuffix(s, suffix string) bool: 判断后缀
  • strings.Contains(s, substr string) bool: 包含字符
  • strings.Index(s, str string) int: str在s中的索引,返回-1代表要查询的字符串不存在
  • strings.LastIndex(s, str string) int: str在s中的最后一次的索引,返回-1代表要查询的字符串不存在
  • strings.IndexRune(s string, r rune) int:非ASCII编码的字符，建议使用该函数来对字符进行定位
  • strings.Replace(str, old, new, n) string: Replace 用于将字符串 str 中的前 n 个字符串 old 替换为字符串 new，并返回一个新的字符串，如果 n = -1 则替换所有字符串 old 为字符串 new
  • strings.Count(s, str string) int: Count 用于计算字符串 str 在字符串 s 中出现的非重叠次数
  • strings.Repeat(s, count int) string: Repeat 用于重复 count 次字符串 s 并返回一个新的字符串
  • strings.ToLower(s) string: 将字符串中的 Unicode 字符全部转换为相应的小写字符
  • strings.ToUpper(s) string: 将字符串中的 Unicode 字符全部转换为相应的大写字符
  • strings.TrimSpace(s): 来剔除字符串开头和结尾的空白符号
  • strings.Trim(s, “cut”): 将开头和结尾的 cut 去除掉
  • TrimLeft/TrimRight: 只想剔除开头或者结尾的字符串
  • strings.Fields(s):利用个空白符号来作为动态长度的分隔符将字符串分割成若干小块，并返回一个 slice
  • strings.Split(s, sep): 用于自定义分割符号来对指定字符串进行分割，同样返回 slice
  • strings.Join(sl []string, sep string) string: Join 用于将元素类型为 string 的 slice 使用分割符号来拼接组成一个字符串

mysql总结

发表于 2017-01-23   |     |   阅读次数

系统级命令

  • mysql -u root -p
  • brew services start mysql:启动mysql服务
  • brew services stop mysql:停止mysql服务
  • exit:退出mysql环境

SQL基础语句

DDL

  • CREATE DATABASE databasename- 创建新数据库
  • ALTER DATABASE - 修改数据库
  • CREATE TABLE tablename- 创建新表
  • DROP TABLE tablename - 删除表格
  • ALTER TABLE - 变更（改变）数据库表
  • DROP TABLE - 删除表
  • CREATE INDEX - 创建索引（搜索键）
  • DROP INDEX - 删除索引

DML

  • 增
    INSERT INTO Persons VALUES (2, ‘Gates’, ‘Bill’, ‘Xuanwumen 10’, ‘Beijing’)

  • 删
    DELETE FROM 表名称 WHERE 列名称 = 值
    DELETE FROM Person WHERE LastName = ‘Wilson’
    DELETE FROM table_name or DELETE * FROM table_name:可以在不删除表的情况下删除所有的行。这意味着表的结构、属性和索引都是完整

  • 改
    UPDATE 表名称 SET 列名称 = 新值 WHERE 列名称 = 某值
    UPDATE Person SET FirstName = ‘Fred’ WHERE LastName = ‘Wilson’

  • 查
    *代表所有的列
    SELECT username FROM userinfo

where
SELECT * FROM Persons WHERE City=’Beijing’

and
select * from userinfo where username=’fzh’ and departname=’dev’

distinct
SELECT DISTINCT Company FROM Orders :去重查找

ORDER BY :默认为升序，DESC为降序
select username from userinfo order by created

SQL高级语句

  • top:限制返回数据的数量
    SELECT username FROM userinfo LIMIT 3

  • LIKE 操作符用于在 WHERE 子句中搜索列中的指定模式
    select * from userinfo where username like ‘f%’

通过使用 NOT 关键字来表示否
select * from userinfo where username not like ‘%a%’

  • IN 操作符允许我们在WHERE子句中规定多个值
    select * from userinfo where username in (‘aaa’,’fzh’)

  • BETWEEN … AND 会选取介于两个值之间的数据范围。这些值可以是数值、文本或者日期
    select username from userinfo where username between ‘aaa’ and ‘bbb’

请使用 NOT 操作符来表示否
select username from userinfo where username not between ‘aaa’ and ‘bbb’

  • NOT NULL 约束强制字段始终包含值。这意味着，如果不向字段添加值，就无法插入新记录或者更新记录

  • UNIQUE 约束唯一标识数据库表中的每条记录,每个表可以有多个 UNIQUE 约束，但是每个表只能有一个 PRIMARY KEY 约束

相关配置文件说明

  • /etc/profile（建议不修改这个文件）全局（公有）配置，不管是哪个用户，登录时都会读取该文件。

  • /etc/bashrc(一般在这个文件中添加系统级环境变量)全局(公有)配置，bash shell执行时，不管是何种方式，都会读取此文件。

  • ~/.bash_profile(一般在这个文件中添加用户级环境变量)每个用户都可使用该文件输入专用于自己使用的shell信息,当用户登录时,该文件仅仅执行一次!

  • 想配置文件立即生效： 运行source .bash_profile

  • 若为bash，在终端输入zsh，则切换为zsh

读书之路-读《蔡康永说话之道上》有感

发表于 2017-01-15   |     |   阅读次数

前言

从学校不如社会，最大的改变就是环境变得更加的复杂。在学校里面与别人交流比较单纯，不用想太多有的没的，但是步入社会之后，你会发现想得简单就会被套路。加上我又是说话比较不经大脑的人，所以经常被套路。希望通过读书多学一些东西，能够跟别人更好的交流，少被套路。

说明

《蔡康永的说话之道》这本书分上下两部，每部定价30元，第一本是总结出40条的核心思想，然后在通过核心语句来扩展描述。我读上部这本书大概花了8个小时左右，读完这本书之后，个人感觉没有值回书钱，感同身受的比较少(纯属个人感觉，不喜勿喷)。下面我将个人觉得比较好的分享一下。

个人感觉不错的章节

  • 外表好不好看，绝对不是人生的决胜点

现在的社会过于浮躁，致使重视心灵和内在的比较少，导致大众过多的重视外貌。所以才导致现在多家整容整形医院的存在，虽然不乏各种整形失败甚至死亡的案例，依然抵挡不住群众整形的热潮。其实当你与别人相处的时候，好的容貌确实能给你带来好的第一印象，但是如果你的性格存在各种缺陷，别人也只是会说，别看那个人长得人模狗样的，但是人品差的一逼。这样的话，别人照样不会和你做朋友的。

  • 沉默是没问题的，沉默是很正常的

你有没有曾经和你不熟的人在一起时，常常陷入没话说的尴尬境界中，因为我本身就是跟熟的的人能胡扯一天，但是跟不熟的人就说不话的人，所以我跟不熟的人坐在一起没话说的时候总会感觉很尴尬。其实想想完全没必要，享受沉默也是不错的，既然双方没有那么熟，何不安静的呆一会。

  • 不想交谈言深的话，应该避开地雷

与刚认识的人或者跟想认识的交流的时候，不太适宜交谈一些损人或者容易触及别人底线的东西。例如书中举得例子，在你上某个教授的课时，你看见邻座的女生非常漂亮，这时候你上去搭讪说:“你看那个教授长得好奇怪，长得跟马铃薯似得”。结果女生说了一句，那是她爸就走了。这他妈的就很尴尬了。所以当你第一次跟别人交流的时候最好说一些不带有攻击性的话题。

  • 交谈不是有奖金的竞赛，别急着抢答

在高手面前，装逼是没有用的，只会让别人看到你更多的缺点，最好的就是珍惜碰到高手的机会，好好地学习高手为什么成为高手。在与别人的交流中，不要打断别人正在讲的话，比如别人正在讲一个你听过的冷笑话，你应该适当的配合一下，而不是将答案脱口而出，我相信这样并不会帮助你成为别人的朋友。(以前我也是爱打断别人说话的人，现在不是了)

  • 初次见面，应该说些什么

正如文中所讲，初次见面，如果对方说了自己的名字，你也应该自报家门，这是最起码的礼貌。

  • 赞赏，观察对方最想被认可的部分

我在很多不同的书上都提到了，赞赏比批评更加有用，你批评别人，只会让别人对此事产生抵触的情绪(当然并不是所有的都不能批评，这是就事而论的)。

  • 别人赞美你一句，你就回一句赞美

  • 不要群发祝福短信

群发短信这种东西并不能巩固你跟接受者的友情，你这样反而会使对方感觉你们的距离更加的遥远。在发祝福短信的时候最起码应该在前面加上对方的名字，让对方知道你不是在敷衍这份关系。

  • 炒热气氛，需要练习(不要在KTV当麦霸)

在有些聚会中，往往有些人会肆无忌惮顶替别人的歌或者拿着话筒一直唱到结束，大家都是来放松的，不是来听你的个人演唱会的。

个人不同意的一些章节

  • 带来惊叹号，就会留下深刻的印象

本章节书中讲到，当所有人都照本宣科的时候，谁能带来惊叹号，谁就能留下深刻的印象，虽然未必是好印象，但比起完全忽略来讲，还是比较好的事吧。
但是这种观点也导致各种标题党横飞，各种无营养的新闻和娱乐人为了赚眼球而搏出位。做人做事，我觉得还是要有点底线和良心的，虽然良心并不能让你赚钱。

总结

任何的说话技巧都不是万能的，面对不同的人要有不同的方式。而且要在现实交流中不断的总结反省自己，只有多和别人交流才能知道自己的不足在什么地方。
用本书的一句话结尾。
我觉得语言最美的时候，就是在我们透过语言感受到彼此相互需要的时候
